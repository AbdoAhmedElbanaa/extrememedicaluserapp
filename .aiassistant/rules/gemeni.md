## 1. Task Completion

* Always complete the full task.
* Never stop mid-implementation.
* Never return incomplete code.

---

## 2. Full Feature Requirement

Every feature MUST include:

* UI
* Logic
* Data Layer
* Integration

If any part is missing → task is incomplete.

---

## 3. Mandatory File Generation

When creating a feature, you MUST generate:

* views
* widgets
* controllers
* models
* services
* data
* docs

No skipping allowed.

---

## 4. Design Rules

* Use ONLY project theme system
* No hardcoded values
* Follow spacing & typography system

---

## 5. Responsiveness Enforcement

* UI must adapt to:

    * Mobile
    * Tablet
    * Desktop/Web

No fixed layouts allowed.

---

## 6. Error Handling (Required)

* Wrap risky operations in try/catch
* Provide user-friendly error messages
* Log errors

---

## 7. Validation Before Output

Before returning ANY result:

* Check for syntax errors
* Check imports
* Check logic consistency

If any issue found:

* Fix it BEFORE returning

---

## 8. Retry Logic

If task fails:

1. Retry automatically
2. Attempt alternative solution
3. If still failing → explain clearly + provide workaround

---

## 9. No Silent Stops

You are NOT allowed to:

* Stop without explanation
* Say "cannot complete" without solution

You MUST:

* Explain issue
* Suggest fix
* Provide partial working alternative

---

## 10. Code Quality Enforcement

* Clean architecture
* Modular structure
* Reusable components
* No duplicated code

---

## 11. Output Format

* Always structured
* Always organized
* Always production-ready

---

## 12. Final Rule

If the feature is not:

* working
* responsive
* structured
* validated

Then it is NOT complete.
